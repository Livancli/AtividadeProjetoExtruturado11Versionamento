QUESTÕES

PARA QUE SERVE O COMMIT E QUAL A MELHOR FORMA DE UTILIZÁ-LO?

Em sistemas de controle de versão como o Git, um commit adiciona as alterações mais recentes do código-fonte para o repositório, tomando essas alterações parte da revisão principal do repositório, o commit vem após o uso da funções "add", que adiciona os arquivos selecionados na função em uma área de espera, a versão do arquivo fica temporariamente salva, mas não de forma definitiva, sendo que se você utilizar a função "add" mais uma vez para o mesmo arquivo, ele é sobrescrito. Para "commitar" no git, basta executar o comando: git commit -m "mensagem", depois que o commit foi aplicado, a última etapa é enviar o commit para o repositório: git push origin 'nome da sua branch'.

QUAL A DIFERENÇA ENTRE COMMIT E PUSH?

O commit realiza a gravação dos arquivos no repositório local, e o push faz a gravação no repositório remoto, com o push é possível a verificação em qualquer estação e no commit a alteração não estará disponível em outros lugares, o commit é usado em conexão com seu repositório local e o outro é usado para interagir com um repositório remoto.

O QUE É TAG?

As tags no git funcionam como marcadores para commits específicos, são como branches, exceto que tags são deletadas, a tag permite que você adicione informações sobre um commit, ou divida todos os seus commits, em grandes grupos, tem dois tipos de tags: annotated e non-annotated. Ao usá-las, existe uma pequena diferença entre elas. Ambas permitem que você faça referência a um commit específico em um repositório. Uma annotated tag cria uma objeto tag adicional no repositório do Git, o que permite que você armazene informações associadas com a tag. Isso pode incluir notas de liberação, a meta-informação sobre o lançamento e, opcionalmente, uma assinatura para verificar a autenticidade do commit para o qual ele aponta.

O QUE É BRANCH?

Uma ramificação no git é um ponteiro para as alterações feitas nos arquivos do projeto, os repositórios no GitHub funcionam como uma árvore, isso porque o branch permite que com a mesma pasta de arquivos, você possa ter várias áreas de trabalho, ou ambientes diferentes, é útil em situações nas quais você deseja adicionar um novo recurso ou corrigir um erro, gerando uma nova ramificação garantindo que o código instável não seja mesclado nos arquivos do projeto principal. Depois de concluir a atualização dos códigos da ramificação, você pode mesclar a ramificação com a principal, geralmente chamada de master.

COMO RESOLVER POSSÍVEIS CONFLITOS NO GIT?

Como qualquer linguagem ou tecnologia, estamos sujeitos a nos depara com conflitos, conflitos podem acontecer tanto ao mesclar branches, bem como quanto ao mesclar commits dentro da mesma branch, são bastantes comuns e acontecem sempre quando o mesmo arquivo foi modificado por duas versões diferentes e essas versões não podem ser automaticamente mescladas, é preciso localizar todos os arquivos onde ocorreram conflitos para a resolução do conflito.
